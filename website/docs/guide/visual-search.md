# 自然语言搜索

自然语言搜索会为图片建立本地视觉索引，让你可以用“红色包装海报”“极简登录页”“户外咖啡照片”这类描述找回素材。

这个功能在本机运行，不会把图片上传到外部服务。首次使用前需要准备视觉模型目录。

## 准备模型

拾光当前支持两类本地视觉搜索模型：

- [FG-CLIP2 ONNX](https://huggingface.co/zihuv/fg-clip2-base-onnx)
- [Chinese-CLIP ONNX](https://huggingface.co/zihuv/chinese-clip-vit-base-patch16-onnx)

两者都可以使用，推荐优先从你已经整理好的 Hugging Face 模型仓库下载，并保持整个模型目录结构完整，不要只单独下载某个 `onnx` 文件。

下载后按下面步骤处理：

1. 解压模型包到一个固定位置，例如 `~/Documents/shiguang-models/fgclip2_flat`
2. 打开拾光的“设置 -> AI -> 本地自然语言搜索”
3. 在“模型目录”里选择解压后的模型文件夹
4. 点击“校验目录”，看到模型状态可用后再启用自然语言搜索

建议不要把模型目录放进临时下载目录。后续应用会记住这个路径，如果目录被移动或删除，需要重新选择。

## 选择哪个模型

- `FG-CLIP2` 更适合复杂画面、主体关系和更细粒度的视觉语义检索。
- `Chinese-CLIP` 更轻量，适合先快速体验本地自然语言搜索。
- 如果你主要用中文描述搜图，并希望先顺利跑通流程，可以先试 `Chinese-CLIP`；如果你更在意复杂场景的找图效果，可以优先试 `FG-CLIP2`。

## 模型目录要求

拾光选择的是“模型目录”，不是单个模型文件。所选目录至少需要包含：

- `model_config.json`
- 文本编码模型文件
- 图像编码模型文件
- 分词器文件

其中 `FG-CLIP2` 还需要额外的 token embedding 和 vision position embedding 文件。如果目录不完整，设置页会在“模型状态”里提示缺少哪些文件。

只要目录里有 `model_config.json`，并且相关依赖文件齐全，就可以直接在拾光里点击“校验目录”确认是否可用。

## 自动识别

启动时，拾光会尝试识别一些常见模型目录：

- 当前工作目录下的 `models/fgclip2_flat`
- 当前工作目录下的 `models/chinese_clip_flat`
- 开发调试目录 `.debug-models/fgclip2_flat`
- 开发调试目录 `.debug-models/chinese_clip_flat`

如果模型已经放在这些位置，设置页会自动填入可用路径。没有自动识别到时，手动选择目录即可。

## 使用步骤

1. 下载并解压上面的任意一个模型目录
2. 在拾光设置中打开“AI -> 本地自然语言搜索”
3. 选择模型目录并点击“校验目录”
4. 校验通过后启用“自然语言搜索”
5. 点击“处理未索引图片”完成首次索引
6. 回到素材页，直接输入“黑底海报”“海边情侣背影”“绿色包装设计”这类描述开始搜索

## 建立索引

模型校验通过后，可以在设置页点击“处理未索引图片”。首次处理会花一些时间，后续新增图片只需要处理增量内容。

如果开启“导入后自动建立视觉索引”，新导入的图片会在后台排队处理。

如果你切换了模型，建议重新处理一次未索引图片，让新模型生成一致的向量索引。

## 常见问题

### 为什么提示未找到 `model_config.json`

通常是因为选中了上一级目录，或者只下载了仓库中的单个文件。请重新选择解压后的模型根目录。

### 为什么校验通过了但搜不到结果

先确认已经执行过“处理未索引图片”。模型可用只代表推理环境正常，图片还需要先建立索引才能被自然语言搜索命中。

### 可以把模型放到哪里

建议放在固定目录，例如 `~/Documents/shiguang-models/`。不要放在系统下载临时目录、移动硬盘临时挂载目录，或经常清理的缓存目录里。

## 使用建议

自然语言搜索适合查找视觉特征、画面内容和风格相近的图片。文件名、标签、颜色筛选仍然有价值，建议和自然语言搜索一起使用。
