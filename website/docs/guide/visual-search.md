# 自然语言搜索

用"红色包装海报"、"极简登录页"、"户外咖啡照片"这类描述来找图。所有计算在本机完成，不上传图片到外部服务。

首次使用前需要准备视觉模型目录。

## 准备模型

拾光支持两类本地模型：

- [FG-CLIP2 ONNX](https://huggingface.co/zihuv/fg-clip2-base-onnx) — 更适合复杂画面和细粒度检索
- [Chinese-CLIP ONNX](https://huggingface.co/zihuv/chinese-clip-vit-base-patch16-onnx) — 更轻量，适合快速体验

下载后：

1. 解压到固定位置，例如 `~/Documents/shiguang-models/fgclip2_flat`
2. 打开设置 → AI → 本地自然语言搜索
3. 选择模型目录，点击"校验目录"
4. 校验通过后启用自然语言搜索

## 选哪个模型

- 主要用中文描述搜图，想先跑通流程：先试 Chinese-CLIP
- 在意复杂场景的找图效果：优先试 FG-CLIP2

## 模型目录结构

选择的是**模型目录**，不是单个文件。目录至少包含：

- `model_config.json`
- 文本编码模型文件
- 图像编码模型文件
- 分词器文件

FG-CLIP2 还需要额外的 token embedding 和 vision position embedding 文件。如果目录不完整，设置页会提示缺少哪些文件。

## 使用步骤

1. 下载并解压模型
2. 设置中选择模型目录并校验
3. 校验通过后启用自然语言搜索
4. 点击"处理未索引图片"完成首次索引
5. 回到素材页，直接输入描述开始搜索

## 建立索引

模型校验通过后，在设置页点击"处理未索引图片"。首次处理需要一些时间，后续新增图片只需处理增量。

开启"导入后自动建立视觉索引"，新导入的图片会在后台自动处理。

切换模型后，建议重新处理一次未索引图片。

## 常见问题

### 找不到 model_config.json

通常是因为选错了目录，或者只下载了单个文件。重新选择解压后的模型根目录。

### 校验通过但搜不到结果

先确认是否已执行"处理未索引图片"。模型可用不代表图片已建立索引。

### 模型放哪里合适

固定目录，例如 `~/Documents/shiguang-models/`。不要放在系统下载目录或经常清理的缓存目录里。
