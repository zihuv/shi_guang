# 核心能力

拾光不是单纯把图片列出来，而是把素材库里最常见的几个动作接到一起：收集、整理、筛选、找回。

## 多目录索引

你可以把多个本地目录加入素材库，让它们在同一个界面里被浏览和管理。

适合下面这类情况：

- 旧项目素材散落在不同硬盘或不同工作目录
- 下载目录里临时收了一堆图，之后还想继续归档
- 想把“项目素材”和“长期灵感库”放进同一套视图里

## 多种导入入口

拾光支持几种比较贴近日常工作的导入方式：

- 文件选择器批量导入
- 拖拽导入
- 剪贴板粘贴图片
- 浏览器扩展采集网页图片

这意味着你不需要等到“整理好了再收进去”，可以先低成本把素材聚合起来。

## 文件夹与标签并行

文件夹适合表达物理归属，标签适合表达语义。

举例来说：

- 文件夹可以按项目、来源或时间分层
- 标签可以标记风格、题材、颜色倾向、可复用场景

两者并行，比只靠目录层级更适合素材库这类会不断增长的内容。

## 元数据补充

除了文件本身，拾光还支持给素材补充额外信息：

- 标签
- 评分
- 备注
- 来源链接

这些信息的价值在于，你以后回来看时，仍然知道它为什么被保存下来。

## 搜索与筛选组合

当素材开始增多后，最重要的能力不是“继续收”，而是“能不能在一分钟内找回”。

当前可以组合使用的查找维度包括：

- 关键词
- 文件夹
- 标签
- 文件类型
- 主色

这类筛选尤其适合处理参考图、配色灵感、竞品截图等高重复内容。

## 后台导入与持续整理

批量导入任务会显示进度，你可以在导入过程中继续浏览当前素材库，不必停下来等任务全部完成。

这对“边整理边收集”的工作方式很重要，尤其是一次性拖入很多文件时。

## 删除模式与回收站

素材整理过程中，误删通常比“多留一阵子”更麻烦。

拾光支持：

- 回收站模式
- 永久删除模式

如果你正在做大规模清库，建议优先开启回收站模式。

## 外观与效率项

当前应用里还包含一些效率型设置：

- 浅色 / 深色主题切换
- 快捷键配置
- 索引重建

这些能力不算花哨，但对长期使用的手感有帮助。
