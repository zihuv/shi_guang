# Changelog

本文档记录所有值得用户关注的变更。

格式参考 [Keep a Changelog](https://keepachangelog.com/zh-CN/1.1.0/)，版本号遵循 [Semantic Versioning](https://semver.org/lang/zh-CN/)。

## [Unreleased]

### Changed

- `PDF` 和 `PSD` 在应用内预览时统一显示快照缩略图，并在预览栏标记“快照缩略图”。
- 缩略图策略收敛为单一 768px WebP 缓存，列表卡片与详情侧栏共用同一张缩略图，并统一写入 `.shiguang/thumbs/`。
- `PDF` 和 `PSD` 缩略图改为应用内自带渲染依赖，打包后可直接使用，无需用户额外安装系统工具。

### Fixed

- 仅为视频、超大图片和达到阈值的素材生成缩略图，普通小图直接回退原图，减少不必要的缓存和重复生成。
- 导入与重建索引后的缩略图生成改为后台异步执行，并基于内容哈希复用缓存，降低主流程阻塞和重复转码。
- 修复开发模式下移除菜单栏后按 `F12` 无法打开开发者工具的问题。
- 修复发布脚本更新版本号时遗漏 `package-lock.json`，避免发布提交中的 npm 锁文件版本不同步。
- 修复素材外部拖拽时图标异常偏大，并限制详情预览页不再支持拖拽到文件夹或应用外部。
- 修复素材卡片缺少内部拖拽源，恢复拖拽到文件夹时的移动操作。
- 修复单张素材在网格和自适应视图中过度放大占满内容区的问题，改为限制宽度并居中展示。
- 修复素材列表快速滚动时缩略图加载滞后，减少滚动后长时间显示占位图的情况。
- 为素材卡片增加 ThumbHash 占位图，并在旧素材首次显示时自动补齐，减少快速滚动时的黑块和空白感。
- 修复素材卡片加载中 ThumbHash 占位图被文件类型占位覆盖的问题。
- 调整素材列表的虚拟窗口和预热策略，优先保留当前滚动方向附近的缩略图，减少快速上下滑动时反复回退到占位图。

## [0.3.1] - 2026-04-21

### Added

- 新增 Electron 桌面运行时，素材库、导入、扫描、缩略图、文件夹、标签、回收站、浏览器采集和 AI 元数据分析可在 TypeScript 后端中运行。
- 新增 Electron 版本本地 `chinese_clip` 自然语言搜图链路，支持模型目录校验、CPU 推理、视觉索引建立、导入后自动向量化和文本搜图。
- 本地自然语言搜图新增 `fg_clip` 模型目录支持，可使用 FG-CLIP2 建立视觉索引和执行文本搜图。
- 本地自然语言搜图新增平台 GPU 加速选择，Windows 可使用 DirectML，Apple Silicon 可使用 CoreML；运行时保持懒加载，不影响应用启动速度。

### Changed

- 桌面端构建发布链路切换为 npm、electron-vite、Vite 8、Electron 和 electron-builder。
- 本地自然语言搜图状态页现在会显示模型校验、运行时加载状态，以及已索引、失败、待处理、过期图片数量。
- 发布脚本现在会把 `Unreleased` 变更整理为新版本 changelog，并写入 GitHub Release 说明。

### Fixed

- 隐藏 Windows 默认菜单栏，并修正桌面窗口与拖拽使用的应用图标资源。
- 修复 FG-CLIP2 处理小尺寸图片时 patch bucket 和位置向量长度不一致导致索引失败的问题。
- 修复 Electron 发布页下载链接与实际产物不一致的问题，并在发布流程中校验平台原生推理依赖。
- 修复 Linux `.deb` 打包缺少维护者邮箱导致发布流程失败的问题。

### Removed

- 移除前端运行时对 Tauri IPC、文件系统、日志和对话框插件的依赖。
- 移除 Tauri/Rust 后端和对应的构建发布流程。
